# 产品项目工作区

本目录用于承载一个新产品从 0 到 1 的完整设计与推进过程，覆盖业务分析、需求管理、产品方案、版本规划、数据指标、风险决策与设计协同。

## 目录说明

- `00-项目总览`：项目导航、范围、关键结论与对外统一口径
- `01-战略与背景`：项目背景、问题定义、业务目标、价值判断
- `02-用户与场景`：用户角色、画像、核心场景、用户旅程
- `03-业务流程`：现状流程、目标流程、跨部门协作流程
- `04-需求管理`：需求池、需求分析、优先级管理
- `05-产品方案`：PRD、信息架构、功能设计说明
- `06-版本规划`：版本节奏、里程碑、上线范围管理
- `07-数据与指标`：目标指标、过程指标、分析口径
- `08-风险与决策`：风险清单、边界管理、关键决策记录
- `09-设计与原型`：设计说明、原型链接、交互补充说明
- `10-会议与调研`：访谈记录、会议纪要、调研沉淀
- `99-附件与参考`：外部资料、行业参考、补充附件

## 推荐使用顺序

1. 先填写项目背景与业务目标
2. 再明确用户角色、核心场景与现状流程
3. 基于问题与流程沉淀需求池
4. 输出产品方案 PRD 与版本规划
5. 同步建立指标体系、风险清单与决策记录

## 建议规则

- 任何关键结论优先落在文档中，不只停留在沟通里
- 需求变更必须更新 `04-需求管理` 与 `08-风险与决策`
- 版本承诺必须同步更新 `06-版本规划`
- 设计链接、会议结论、调研证据要保留来源

## 目录说明

- `00-项目总览/` 到 `10-会议与调研/`
  项目策划与过程资料。
- `99-附件与参考/`
  附件和参考内容。
- `portfolio-src/`
  作品集源文件。
- `portfolio-build/`
  作品集构建产物与预览资源。
- `ui-redesign/`
